Pipeline Stages
1
History
Processes Gmail push notifications via Gateway history endpoint. Filters for new messages, routes SENT messages to fast-path, and produces INBOX messages to the fetch stage.
2
Fetch
Fetches full thread content from Gmail Gateway with summarization. Cleans HTML, filters SPAM/DRAFT, detects corporate and excluded senders, extracts all sender emails.
3
Enrich
hubspot-api
Looks up each customer in HubSpot via unified /lookup endpoint. Generates LLM customer summary with deal history, collects suggested labels from orders, RDM cases, and payment status.
4
Classify
gpt-4.1-mini
Classifies email using OpenAI Structured Outputs: category, priority, sentiment, action required, and follow-up type. Merges programmatic labels with HubSpot suggested labels.
5
Draft
gpt-5-mini
Composes context-aware draft replies using customer history, classification, and HubSpot data. Creates threaded Gmail drafts with proper In-Reply-To headers. Skips when no reply is needed.
6
Complete
Applies Gmail labels (4–13 per thread), handles conflicting label removal, creates labels with color coding via Gmail API, and marks the pipeline as done.
Parallel Outputs
Pinecone
text-embedding-3-small
Generates embeddings and upserts to Pinecone for semantic email search. Filters spam and low-quality emails. Updates vectors when threads evolve or replies are sent.
Monday Tasks
monday-api
Creates follow-up tasks on Monday.com boards with customer details, HubSpot links, Gmail thread links, and RDM case references. Auto-marks tasks Done when reply is sent.

Sent Message Fast-Path
SENT Detection
stage-sent-mail-complete
When a team member sends a reply, Gmail push notification triggers. History consumer detects the SENT label and routes directly to the fast-path topic — bypassing the full pipeline.
Auto-Resolution
Monday consumer marks the follow-up task as Done. Pinecone consumer replaces the draft text with actual sent content. Complete consumer applies the Action/Completed label to the Gmail thread.

Streaming & Resilience
RedPanda
Kafka-compatible
7 pipeline topics + 6 dead-letter queues. Avro serialization with Schema Registry. Manual offset commits — failed messages retry with exponential backoff (max 3, 5s–300s). Poison messages route to per-stage DLQ.

Backfill & Reprocessing
Idempotent
Reprocess historical emails by date range, mailbox, and starting stage. Force mode clears existing metadata for full re-run. Per-stage idempotency — re-runs safely skip completed messages unless forced.
