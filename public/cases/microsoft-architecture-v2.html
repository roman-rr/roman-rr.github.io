FastAPI Backend

The central orchestrator. Runs on Azure App Service or Container Apps. Handles custom business logic, routes requests to Graph, triggers AI pipelines, and exposes endpoints for frontends and Logic Apps.

Authenticates via MSAL for Python against Entra ID to obtain tokens for Microsoft Graph calls.

Python 3.11+ FastAPI MSAL Azure App Service Docker

Microsoft Graph

Unified REST API layer for all M365 services. Provides read/write access to SharePoint files, Outlook mail, Teams channels, calendar, users, and Business Central data through a single authenticated endpoint.

FastAPI calls Graph endpoints using OAuth2 tokens. Supports delta queries for incremental sync and change notifications via webhooks.

REST API OAuth 2.0 Webhooks Delta Queries

Logic Apps (Workflows)

Low-code automation for event-driven workflows. Triggers on new Outlook mail, SharePoint file uploads, Teams messages, or scheduled intervals. Calls FastAPI endpoints for AI processing, classification, and routing.

Connectors for 400+ services. Handles approval flows, notifications, and cross-system orchestration without custom code.

Azure Logic Apps Connectors HTTP Actions Triggers

RAG Pipeline

Retrieval-Augmented Generation for document intelligence. Documents from SharePoint and Outlook are extracted, split into semantic chunks, embedded into vectors, and stored in a search index.

At query time, relevant chunks are retrieved via hybrid search (vector + keyword) and passed as context to the LLM for grounded, accurate answers.

Azure AI Search Embeddings Chunking Hybrid Search LLM

Copilot Studio Agent

Custom AI agent built in Microsoft Copilot Studio. Connects to Azure AI Search as its grounding knowledge source, enabling the agent to answer questions based on indexed organizational content with citations.

FastAPI endpoints are registered as plugins, allowing the agent to perform actions — create tasks, send notifications, query Business Central, or trigger Logic App workflows — all from natural language conversation.

Copilot Studio AI Search Plugin FastAPI Actions M365 Copilot Teams Web Chat

Microsoft Fabric

Unified analytics platform combining lakehouse, data warehouse, ETL pipelines, notebooks, and Power BI into a single managed environment. Serves as the central data and reporting layer for the entire solution.

Operational data from all integrated systems flows into OneLake for long-term storage, transformation, and business intelligence through Power BI dashboards.

OneLake Lakehouse SQL Warehouse Power BI Spark

Knowledge source — Copilot Studio connects directly to Azure AI Search as a grounding data source. The agent retrieves relevant chunks at query time and generates cited, contextual answers.
Custom actions — The agent can call FastAPI endpoints as plugins to perform actions: create tasks, send emails, update SharePoint lists, or trigger Logic App workflows — all from natural language.
Multi-channel — A single Copilot Studio agent deploys to M365 Copilot (inline in Word, Outlook, Teams), as a Teams bot, as a SharePoint web part, or as a standalone web chat widget — same knowledge, same actions, multiple surfaces.
Always current — Because the event-driven pipeline continuously indexes new content, the Copilot agent always queries the latest knowledge without manual re-indexing or batch schedules.
KEDA Auto-Scaling — Per Worker
Service Bus Subscription
Each domain worker has its own subscription with filtered messages
KEDA Scale Rule
Monitors subscription message count independently per worker
Independent Scaling
Document worker may scale to 5 replicas while email worker stays at 1
Scale to Zero
Idle workers drop to 0 replicas — no cost until next event arrives
Domain isolation — Each worker is a separate Container App with its own codebase, scaling rules, and deployment lifecycle. A bug in the email worker cannot affect document processing.
Parallel processing — A single Graph change notification publishes once to the topic. All matching subscriptions receive it simultaneously, enabling parallel domain processing with no duplication logic.
Durability & retry — Service Bus provides at-least-once delivery with configurable retry policies and dead-letter queues per subscription. Failed messages are preserved for inspection and reprocessing.

Idempotency & Document Metadata Tracking

Every processed event is tracked in Azure Cosmos DB (NoSQL API) to guarantee exactly-once processing semantics and provide a complete audit trail of all pipeline activity.

Why Azure Cosmos DB
Sub-millisecond lookups — Idempotency checks happen on every incoming message. Cosmos DB delivers single-digit-ms point reads by partition key, keeping worker latency low even under burst load.
Schema-flexible documents — Metadata shapes vary across document types (emails, files, messages, meetings). Cosmos DB's schemaless model handles this naturally without migrations.
KEDA-friendly — No connection pool exhaustion under scale-to-zero/scale-up patterns. The HTTP-based SDK handles ephemeral workers cleanly, unlike SQL connection pooling.
Built-in TTL — Processing records can auto-expire after a configurable retention window, keeping storage costs bounded without manual cleanup jobs.
Flow Idempotent Message Processing
01
Receive
Worker pulls message from Service Bus subscription
02
Check
Point read in Cosmos DB by idempotency key — if exists and status = completed, skip
03
Claim
Upsert record with status = processing and a lease timestamp to prevent duplicate work
04
Process
Execute domain logic — chunk, embed, classify, route, or index
05
Commit
Update record to status = completed with output metadata and processing duration
06
Ack
Complete the Service Bus message — removed from queue only after successful commit
Schema Cosmos DB Document Structure
Processing Record
{
  "id":              "graph-notif-a1b2c3d4",
  "partitionKey":    "documents-worker",
  "source":          "sharepoint",
  "resourceId":      "/sites/ops/docs/Q2-report.pdf",
  "changeType":      "updated",
  "graphTimestamp":   "2026-03-10T14:22:00Z",

  "status":          "completed",
  "claimedAt":       "2026-03-10T14:22:01Z",
  "completedAt":     "2026-03-10T14:22:08Z",
  "durationMs":      7042,
  "workerInstance":  "documents-worker-replica-3",

  "output": {
    "chunksCreated":  24,
    "indexName":      "corporate-knowledge",
    "embeddingModel": "text-embedding-3-large",
    "contentHash":    "sha256:e3b0c442..."
  },

  "ttl":             2592000
}
Partition key Partition by worker name — each domain worker queries only its own records, ensuring hot partition isolation and efficient point reads.
Idempotency key The id is derived from the Graph change notification ID. Duplicate deliveries from Service Bus resolve to the same document — Cosmos upsert is atomic.
Content hash SHA-256 hash of the source content. If a file is modified but produces the same hash, re-indexing is skipped — saving embedding and AI Search write costs.
Lease & timeout If a worker crashes mid-processing, the claimedAt timestamp acts as a lease. A background sweep or the next delivery reclaims stale records after a configurable timeout (e.g. 5 minutes).
TTL auto-expiry Records expire after 30 days (2592000 seconds) by default. Adjustable per domain — email records may retain longer for audit compliance.
Observability Processing duration, chunk counts, and error fields feed into Fabric analytics for pipeline health dashboards — latency trends, failure rates, throughput by domain.

Use Case Examples

Practical scenarios showing how the architecture components work together to solve real business problems.

RAG + Graph

Company Knowledge Assistant

Employees ask natural-language questions about internal policies, contracts, or product specs. The system retrieves relevant SharePoint documents via Graph, chunks and embeds them, then delivers precise, cited answers through a Teams bot or web chat.

SharePoint→ Graph API→ FastAPI→ RAG Pipeline→ Teams Bot
Logic Apps + FastAPI

Intelligent Email Triage

Incoming client emails in Outlook trigger a Logic App that calls FastAPI. The AI classifies intent (support request, invoice, complaint, inquiry), extracts key entities, drafts a response, and routes to the correct Teams channel or SharePoint list. Urgent items get flagged with priority notifications.

Outlook→ Logic App→ FastAPI + LLM→ Teams / SharePoint
Fabric + Power BI

Business Central Analytics Dashboard

Sales orders, inventory levels, and financial data from Business Central flow into Microsoft Fabric's lakehouse via scheduled pipelines. Power BI dashboards surface real-time KPIs — revenue trends, margin analysis, fulfillment rates — accessible to leadership directly in Teams tabs.

Business Central→ Graph API→ Fabric Lakehouse→ Power BI
Graph + Logic Apps

Client Onboarding Automation

When a new client record is created in the CRM, a Logic App triggers an automated onboarding workflow: creates a dedicated SharePoint site with template documents, sends a welcome email sequence via Outlook, schedules a kickoff meeting in calendar, and posts an intro in the team's channel.

CRM Trigger→ Logic App→ Graph API→ SharePoint + Outlook + Teams
RAG + Fabric

Compliance & Contract Review

Legal and compliance documents stored in SharePoint are continuously ingested, chunked, and indexed. Staff can query across thousands of contracts — "which agreements expire in Q2?" or "find all clauses related to data retention." Analytics on query patterns and document usage flow to Fabric for compliance reporting.

SharePoint Docs→ RAG Index→ FastAPI Query→ Fabric Reports
Full Stack

Meeting Intelligence Hub

Teams meeting transcripts and recordings are processed by FastAPI — summarized, action items extracted, and decisions logged. Summaries are posted back to the meeting chat, action items create tasks in Planner via Graph, and meeting analytics (frequency, topics, participation) are tracked in Fabric dashboards.

Teams Meetings→ FastAPI + LLM→ Graph (Planner)→ Fabric Analytics

Technical Highlights

Cloud-Native & Scalable

The entire platform runs on Azure managed services — App Service, Container Apps, Logic Apps, Fabric — minimizing infrastructure overhead while allowing independent scaling of each layer as demand grows.

AI-Powered Document Intelligence

Documents and emails are automatically chunked into semantic passages, embedded as vectors, and indexed for hybrid search — enabling precise, grounded answers from your organization's knowledge base.

Secure Identity & Access

All service-to-service communication is authenticated through Microsoft Entra ID (Azure AD) with OAuth 2.0 tokens via MSAL. Role-based access control ensures data stays within organizational boundaries.

Unified Data & Analytics

Microsoft Fabric consolidates data engineering, warehousing, and BI into a single platform — operational data flows into OneLake and surfaces as Power BI dashboards with minimal pipeline complexity.
